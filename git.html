git / Github

Git

Git er et versjonskontrollsystem. Et versjonskontrollsystem er et program som holder styr på endringer som blir gjort på en fil. Med git kan man gå tilbake til en eldre versjon av filen hvis man vil. Man kan se hvem som har gjort endringer, og hvor endringene er, så med Git er det lettere å samarbeide. I git er det lett å lage en branch og merge den. Noe som er fint med Git er at det er gratis.
En ulempre med Git er at man må lære det. Selv om Git har mange nyttige bruksområder, er det også litt kjipt å måtte lære å bruke det. Alt i alt vil jeg si at Git er et nyttig, om ikke essensielt, redskap for de som koder.

Wikipedia har dette å si om Git:

Git er et versjonskontrollsystem, en type programvare. Det kjennetegnes ved å være distribuert og ikke mappehierarki-basert (se Egenskaper). Et viktig designkriterium for Git var hastighet ved sammenfletting (merging). Git ble påbegynt av Linus Torvalds, men på et tidlig tidspunkt overtatt av Junio Hamano, som nå leder videre utvikling og vedlikehold. Blant andre distribuerte versjonskontrollsystemer kan nevnes Mercurial, som ble startet omtrent samtidig med Git, og Bitkeeper, en kommersiell forløper som var den viktigste inspirasjonskilden for Git.
https://no.wikipedia.org/wiki/Git

GitHub

GitHub er en online tjeneste hvor man kan laste opp filene sine og så har man Git versjonskontroll. GitHub ble opprettet i 2018 og har blitt veldig populært. GitHub og liknende tjenester er gode å ha fordi de har et grafisk brukergrensesnitt, i motsetning til å skrive i terminal. Bildet ovenfor er et screenshot fra Github.
